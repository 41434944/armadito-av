UMWSU-MODULES(3)
================
:doctype: manpage

NOM
---
umwsu-modules - modules de recherche de malwares


SYNOPSIS
--------

enum umwsu_mod_status {
  UMWSUMOD_OK,
  UMWSUMOD_INIT_ERROR,
};

enum umwsu_status {
  UMWSU_CLEAN,
  UMWSU_MALWARE,
  UMWSU_EINVAL,
  UMWSU_IERROR,
  UMWSU_UNKNOWN_FILE_TYPE,
};

typedef enum umwsu_mod_status (*umwsu_mod_init_t)(void **pmod_data);

typedef enum umwsu_status (*umwsu_mod_scan)(const char *path, void *mod_data);

typedef enum umwsu_mod_status (*umwsu_mod_close)(void *mod_data);


DESCRIPTION
-----------
Les modules de recherche sont des objets chargés par la fonction *dlopen*(3). Ils implémentent des fonctions qui sont appelées par la librairie *libumwsu*(3) pour réaliser la recherche de malwares dans des fichiers.

Les modules de recherche sont chargés dynamiquement à partir des fichiers .so situés dans le répertoire */usr/lib/libumwsu/modules* (ce répertoire peut changer en fonction de l'installation). Chaque module exporte certaines fonctions qui seront utilisées par la librairie *libumwsu*. Pour plus d'informations, se référer à *umwsu-modules*(3).

Un module de recherche doit définir les fonctions suivantes (MODULE désigne le nom du module, obtenu à partir du nom du fichier):

*MODULE_init*::
Fonction d'initialisation. Alloue la structure privée du module, initialise le module, retourne la structure allouée via 'pmod_data'.

*MODULE_scan*::
Fonction de recherche. Utilise la structure passée via 'pmod_data' (structure allouée par 'MODULE_init'); réalise la recherche et retourne un statut indiquant si le fichier contient ou non un malware.

*MODULE_close*::
Fonction de dé-initialisation. Libère la structure privée du module, dé-initialise le module.
